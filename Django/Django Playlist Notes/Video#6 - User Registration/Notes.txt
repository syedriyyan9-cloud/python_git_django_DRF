Creating a New "Users" App:

It is best practice to keep user-related logic (like forms and templates) separate from the blog itself.
Create the app using the command python manage.py startapp users.
Important: Always add the new app to the INSTALLED_APPS list in your project's settings.py file immediately so Django recognises it.

Using Django’s Built-in Forms:

Django provides a ready-to-use class called UserCreationForm.
This form handles complex tasks like checking if passwords match, validating email formats, and hashing passwords for security so you don't have to write that code yourself.

Setting up the Register View:

In views.py, you create a function that handles both GET and POST requests.
GET request: When a user first visits the page, they are shown a blank form.
POST request: When a user submits the form, Django validates the data. If it is valid, the user is saved to the database using form.save().

Creating the Template:

Create a register.html file that extends your main base.html.
The form must use the POST method for security.
You must include a {% csrf_token %} inside the form; this is a security tag that protects your site from certain types of attacks.

Flash Messages and Redirects:

Flash Messages: These are one-time alerts (like "Account Created!") that appear once and disappear when the page is reloaded.
Redirects: After a successful sign-up, you can use the redirect function to send the user back to the home page or a login page.
To make messages show up, you must add a small loop in your base template to display any active alerts.

Customising the Form:

To add extra fields (like an email address) that aren't in the default form, you create a new forms.py file.
You create a class that inherits from UserCreationForm and add the new fields you want to include.

Improving Styles with Crispy Forms:

The default Django forms don't look very modern. A popular tool called django-crispy-forms can be installed to style forms automatically using Bootstrap.
After installing it and updating your settings, you simply add |crispy to your form variable in the template to make it look professional and provide clear error messages in red.
