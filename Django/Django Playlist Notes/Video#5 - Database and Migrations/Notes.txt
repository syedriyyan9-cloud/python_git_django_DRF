Understanding the Django ORM: 

Django uses an Object Relational Mapper (ORM), which allows you to interact with your database using Python code instead of writing SQL. 
This makes it easy to switch between different databases, like using SQLite for testing and Postgres for the live website, without changing your code.

Creating a Post Model: 

In Django, database tables are represented by Python classes called models. 
You define a Post model in the models.py file with specific fields, such as a title (limited to 100 characters), content (for long text), and date_posted.

Connecting Authors to Posts: 

You can link posts to users by using a Foreign Key. 
By setting on_delete=models.CASCADE, Django ensures that if a user is deleted, all the posts they wrote are also automatically deleted.

Managing Migrations: 

When you change your models, you must run python manage.py makemigrations to create a plan for the database changes. 
Then, you run python manage.py migrate to apply those changes and create the actual tables in your database.

Using the Django Shell: 

You can interact with your data directly using a command-line "shell" to create, save, and query objects. 
For example, you can use Post.objects.all() to see every post in your database or use filters to find specific users.

Naming Objects Clearly: 

By adding a __str__ method to your model, you tell Django how to display the object (for example, by its title) so it is easier to identify when you are looking at lists of data.

Accessing Related Information: 

Django allows you to easily find all posts written by a specific user using a relationship called post_set. 
You can also access a user's profile information directly from a post, such as getting the author's email address.

Using Real Data in Your Website: 
Instead of using temporary "dummy data," you can import your Post model into your views.py file and query the database to show live posts on your web pages.

Formatting Dates in Templates: 

To make dates look better for users, you can use a date filter in your HTML templates (for example, {{ post.date_posted|date:"F d, Y" }}) to show the full month name and the year.
The Admin Interface: To manage your posts through a graphical dashboard, you must register the Post model in the admin.py file. 
This allows you to create, edit, or delete posts through a simple web interface rather than using code.
